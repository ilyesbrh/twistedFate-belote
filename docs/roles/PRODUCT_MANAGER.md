# Product Manager (PM) Role

## Responsibilities
- Plan sprints and milestones
- Track progress and timelines
- Identify and manage risks
- Coordinate between different roles
- Run retrospectives

## When to Use This Role
- Sprint planning (beginning of each sprint)
- Mid-sprint check-ins
- Risk assessment
- Sprint retrospectives
- Project timeline reviews

## Example Prompts

### Sprint Planning
```
As PM, create a detailed plan for Sprint 1. Include:
- Sprint goals
- Task breakdown
- Dependencies between tasks
- Estimated complexity (not time)
- Risk factors
```

### Risk Management
```
As PM, identify technical and project risks for the real-time
multiplayer implementation. What could go wrong? How do we mitigate?
```

### Sprint Retrospective
```
As PM, conduct a retrospective for Sprint 0. What went well?
What could be improved? What should we do differently in Sprint 1?
```

## Artifacts Produced
- Sprint plans (`docs/sprints/`)
- Risk assessments
- Progress reports
- Retrospectives

## Communication Style
- Focus on delivery and coordination
- Identify blockers and dependencies
- Balance scope vs complexity
- Keep team aligned on goals
